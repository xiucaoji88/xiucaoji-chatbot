# 修草纪智能客服系统部署指南

## 概述
本指南将带你完成：GitHub 托管代码 → Railway 部署 → 微信公众号对接的完整流程。

---

## 第一步：GitHub 托管代码

### 1.1 创建 GitHub 仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `xiucaoji-chatbot` (或你喜欢的名字)
   - **Description**: 修草纪智能客服系统
   - **Visibility**: Private (推荐) 或 Public
   - ✅ 勾选 "Add a README file"
3. 点击 **Create repository**

### 1.2 上传代码到 GitHub

打开终端，执行以下命令：

```bash
# 进入项目目录
cd /Users/xiucaoji/WorkBuddy/20260320194009

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: 修草纪智能客服系统"

# 关联远程仓库 (替换为你的 GitHub 用户名和仓库名)
git remote add origin https://github.com/你的用户名/xiucaoji-chatbot.git

# 推送代码
git branch -M main
git push -u origin main
```

### 1.3 验证上传

访问 `https://github.com/你的用户名/xiucaoji-chatbot` 确认文件已上传。

## 第二步：Railway 部署

### 2.1 注册 Railway 账号

1. 访问 https://railway.app/
2. 点击 "Start for Free"
3. 选择 "Sign up with GitHub" 使用 GitHub 账号登录

### 2.2 创建新项目

1. 登录后点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 点击 "Configure GitHub App" 授权 Railway 访问你的仓库
4. 在列表中找到 `xiucaoji-chatbot` 仓库，点击选择

### 2.3 配置环境变量

项目创建后，点击 "Variables" 标签，添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `WECHAT_APPID` | `wx62ff6236f2c99902` | 公众号 AppID |
| `WECHAT_APPSECRET` | `29bfcf357189861ed2d895970a69fdbf` | 公众号 AppSecret |
| `WECHAT_TOKEN` | `xiucaoji88` | 服务器验证 Token |
| `OPENAI_API_KEY` | `sk-...` | OpenAI API Key |
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | OpenAI API 地址（可选，默认官方） |
| `OPENAI_MODEL` | `gpt-3.5-turbo` | 文本模型（可选，默认 gpt-3.5-turbo） |
| `OPENAI_VISION_MODEL` | `gpt-4o` | 图片识别模型（可选，默认 gpt-4o） |

**API2D 用户注意**：
- API2D 支持的图片识别模型名称可能是 `gpt-4o` 或 `gpt-4-vision-preview`
- 请在 API2D 后台查看支持的模型列表
- 如果图片识别失败，尝试更换模型名称

**企业微信配置（可选，代理专用）**：
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `WORKWECHAT_CORPID` | 企业微信 CorpID | 企业ID |
| `WORKWECHAT_AGENTID` | 应用 AgentID | 自建应用ID |
| `WORKWECHAT_SECRET` | 应用 Secret | 应用密钥 |
| `WORKWECHAT_TOKEN` | 验证 Token | 服务器验证令牌 |
| `WORKWECHAT_ENCODING_AES_KEY` | 加密密钥 | 消息加密密钥 |

添加方式：
- 点击 "New Variable"
- 输入 Name 和 Value
- 点击 "Add"

### 2.4 部署应用

1. 点击 "Deploy" 标签
2. Railway 会自动检测 `Procfile` 并开始部署
3. 等待部署完成（约 1-2 分钟）
4. 部署成功后，点击 "Settings" → "Domains"
5. 你会看到一个默认域名，如 `xiucaoji-chatbot-production.up.railway.app`
6. 复制这个域名，下一步需要用到

### 2.5 验证部署

在浏览器访问：`https://你的域名/health`

如果返回 `{"status": "healthy"}`，说明部署成功！

## 第三步：微信公众号对接

### 3.1 配置服务器 URL

1. 登录微信公众号后台：https://mp.weixin.qq.com/
2. 左侧菜单 → 开发 → 基本配置
3. 点击 "修改配置"
4. 填写以下信息：
   - **URL**: `https://你的域名/wechat`
     - 例如：`https://xiucaoji-chatbot-production.up.railway.app/wechat`
   - **Token**: `xiucaoji88` (与 WECHAT_TOKEN 一致)
   - **EncodingAESKey**: 点击 "随机生成"
   - **消息加解密方式**: 选择 "明文模式"
5. 点击 "提交"

### 3.2 启用服务器配置

1. 配置验证通过后，点击 "启用"
2. 确认启用服务器配置

### 3.3 配置 IP 白名单

1. 在 "基本配置" 页面，找到 "IP 白名单"
2. 点击 "查看"
3. 添加 Railway 服务器的出口 IP
   - 通常可以在 Railway 的 "Settings" → "Networking" 查看
   - 或者先不设置，如果遇到 IP 限制错误再添加

## 第四步：企业微信对接（代理专用）

### 4.1 创建企业微信应用

1. 登录企业微信管理后台：https://work.weixin.qq.com/wework_admin
2. 点击 "应用管理" → "自建" → "创建应用"
3. 填写应用信息：
   - **应用名称**: 修草纪代理助手
   - **应用介绍**: 帮助代理了解产品知识和政策
   - **可见范围**: 选择代理部门或全员
4. 上传应用图标
5. 点击 "创建应用"

### 4.2 获取应用凭证

创建成功后，在应用详情页获取：
- **AgentId**: 应用ID
- **Secret**: 应用密钥（点击查看）

### 4.3 配置接收消息

1. 在应用详情页，点击 "接收消息" → "设置API"
2. 填写以下信息：
   - **URL**: `https://你的域名/workwechat`
   - **Token**: 自定义一个字符串（如 `workwechat88`）
   - **EncodingAESKey**: 点击 "随机生成"
3. 点击 "保存"

### 4.4 配置环境变量

在 Railway 控制台添加企业微信配置：
- `WORKWECHAT_CORPID`: 企业微信 CorpID（在 "我的企业" 页面查看）
- `WORKWECHAT_AGENTID`: 应用的 AgentId
- `WORKWECHAT_SECRET`: 应用的 Secret
- `WORKWECHAT_TOKEN`: 上一步设置的 Token
- `WORKWECHAT_ENCODING_AES_KEY`: 上一步生成的 EncodingAESKey

## 常见问题

### Q1: 服务器配置验证失败？

检查以下几点：
- URL 是否正确（需要 https:// 开头，/wechat 结尾）
- Token 是否与代码中一致
- 应用是否已成功部署并运行
- 域名是否可以正常访问

### Q2: 用户发送消息没有回复？

1. 检查 Railway 日志是否有错误
2. 确认 OpenAI API Key 是否正确
3. 检查微信公众号的 "服务器配置" 是否已启用

### Q3: 如何更新代码？

```bash
# 修改代码后
git add .
git commit -m "更新说明"
git push origin main
```

Railway 会自动重新部署。

### Q4: 如何绑定自定义域名？

1. 在 Railway 控制台 → Settings → Domains
2. 点击 "Custom Domain"
3. 输入你的域名（如 `chat.xiucaoji.top`）
4. 按照提示配置 DNS 解析
5. 在微信公众号后台更新服务器 URL

## API2D 配置 GPT-4 Vision 图片识别

如果你使用 API2D 作为 OpenAI 代理，配置图片识别需要注意：

### 1. 检查 API2D 支持的模型

登录 API2D 后台 https://api2d.com 查看支持的 Vision 模型：
- 常见支持：`gpt-4o`、`gpt-4-vision-preview`
- 在 Railway 环境变量中设置：`OPENAI_VISION_MODEL=gpt-4o`

### 2. 配置环境变量

在 Railway 中添加：
```
OPENAI_API_KEY=你的API2D Forward Key
OPENAI_BASE_URL=https://openai.api2d.net/v1
OPENAI_MODEL=gpt-3.5-turbo
OPENAI_VISION_MODEL=gpt-4o
```

### 3. 测试图片识别

部署后，在公众号发送一张面部照片测试。

### 4. 常见问题

**图片识别返回错误？**
- 检查 API2D 账户余额是否充足（图片识别消耗更多点数）
- 确认 API2D 支持该 Vision 模型
- 查看 Railway 日志获取详细错误信息

## 微店客服对接方案

微店客服系统可以通过以下方式对接：

### 方案 1：客服消息跳转链接（推荐）

在微店后台配置客服消息自动回复，引导用户点击链接进入智能客服：

1. **登录微店后台** → 客服管理 → 自动回复设置
2. **设置欢迎语**：
   ```
   您好！欢迎来到修草纪~
   
   🤖 智能客服：点击咨询 → https://你的域名/chat?source=weidian
   
   💬 人工客服：工作时间 9:00-21:00
   ```

3. **设置关键词回复**：
   - 关键词："祛痘"、"敏感"、"怎么买"
   - 回复：引导点击智能客服链接

### 方案 2：嵌入聊天窗口（需要微店支持）

如果微店支持自定义客服页面：

1. 创建一个简单的聊天页面 `chat.html`
2. 调用 `/api/chat` 接口与 AI 对话
3. 在微店客服设置中嵌入该页面链接

### 方案 3：Webhook 对接（如果微店开放 API）

需要微店提供：
- 客服消息 Webhook 接口
- 发送消息 API

然后类似微信公众号的方式对接。

## 图片识别功能说明

系统已集成 GPT-4 Vision 图片识别功能：

### 用户使用方式
1. 用户发送面部照片到公众号
2. AI 自动分析皮肤问题类型和程度
3. 给出初步判断并引导问诊

### 支持的图片格式
- JPG、PNG 格式
- 建议发送清晰、光线充足的面部照片
- 文件大小不超过 2MB

### 图片分析内容
- 问题类型：痘痘/闭口/敏感/斑点
- 严重程度：轻度/中度/重度
- 分布区域
- 可能成因分析
